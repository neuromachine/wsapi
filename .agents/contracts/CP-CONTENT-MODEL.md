# CP Content Model

## Goal

Create a commercial proposal that:

```text
- explains value quickly
- speaks to a concrete niche and geography
- shows business benefit, not just technical features
- structures choice through packages
- leads to request/contact/discussion
```

## Sections

### 1. HERO

Answers: “Is this about me?”

Fields:

```text
pretitle — 1 short line about market/client behavior
title    — situation/problem
focus    — solution angle / decisive phrase
paragraph — 3–5 concise sentences
```

### 2. Benefits

Answers: “What do I get?”

Use 4–5 items. Each item is a separate business value:

```text
icon
title
text
```

### 3. Extras

Answers: “What changes in numbers/process?”

Use 3–5 practical effects:

```text
more bookings/leads
less manual work
higher average check
faster response time
clearer management
```

### 4. Important

Answers: “How does the system work?”

Use only when needed. Best for:

```text
booking systems
CRM flows
medical platforms
tourism systems
AI assistants
marketplace/catalog automation
```

### 5. Items / Packages

Answers: “What can I choose and how much?”

Usually 3 packages:

```text
Entry / Basic
Growth / Business
System / Premium
```

Rules:

```text
- package 1 = minimum useful launch
- package 2 = recommended balanced offer
- package 3 = complete system/ecosystem
- each next package includes new value, not only more words
```

### 6. Includes

Answers: “What is included in every option?”

Use 3–5 items:

```text
analysis
UX/structure
consultation
basic launch setup
scaling advice
```

## Style

```text
simple > complex
concrete > abstract
business value > technical stack
scannable blocks > long essay
```

## Pricing

Prices must be plausible for WS production work. When unsure, use ranges or “от”.

Do not promise guaranteed metrics unless phrased as projected / based on similar projects.
